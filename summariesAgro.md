Саммарі міту: Розробка ідеї стартапу (Агро-маркетплейс)

Основна ідея: Створення єдиної платформи, що об’єднує агровиробників,
покупців та логістичні компанії для сегментів B2B та B2C.

➢ 1. Функціонал та ролі на платформі

Визначені ролі: Виробник, Покупець, Перевізник, Адміністратор.

Пошукова система: Реалізація фільтрів за товаром, локацією, умовами
продажу, типом закупівлі (опт/роздріб) тощо.

Логістика: Забезпечення доставки товару від виробника до покупця із
залученням зовнішніх компаній-перевізників.

➢ 2. Юридичні та безпекові аспекти

Безпека платежів: Впровадження механізмів гарантування транзакцій.

Договірна база: Розробка користувацької угоди та закріплення
договірних зобов’язань сторін.

Контроль якості: Вирішення питання перевірки товару та визначення
відповідальності обох сторін (продавця та покупця).

➢ 3. Економічна модель та маркетинг

Монетизація: Впровадження абонплати для користувачів та комісії
платформи з угод.

Утримання клієнтів: Пошук рішень проти дезінтермедіації (щоб
користувачі не переходили до прямих угод поза сервісом).

Спеціальні інструменти:

Форвардні закупки:

Об'єднання продавців для виконання великих об'ємів замовлень, шляхом
збору усього об’єму продукції, що є в наявності «в одну складчину».

Просування:
SEO-база, аналітика, створення окремого акаунта у соцмережах (?) та
замовлення мерчу.

➢ 4. Технічний стек

Frontend: Next.js (SSH), TypeScript, Zustand, Tanstack Query, Mantine,
ESLint 9 + Prettier, Formik/Yap (React Hook Form/Zod), Axios, React Hot
Tost;

Backend: Мікросервісна архітектура: Rabbit MQ, JWT, OAuth2, H2,
Hibernate, Java 17, Lombok, Spring Data JPA, Spring Boot, Spring Security,
Spring AI, Mockito, JUnit5, Swagger API, Render;

Database: PostgreSQL, pqAdmin4;

Database Manager: dBeaver;

DevOps: Docker, Docker Compose, AWS, AWS C2;

➢ 5. Наступні кроки:

Окремий міт для Dev-команди: Фіналізація архітектури бекенду.

Загальний мітинг: Заплановано на наступний тиждень для синхронізації
всіх напрямів.
